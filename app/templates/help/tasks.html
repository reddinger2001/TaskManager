TASKS & STATUSES

Statuses

Each task has one status. The workflow flows left to right:

Backlog → Active → Done
Status Meaning
BacklogPlanned but not started yet
ActiveCurrently working on this right now
BlockedCan't proceed until something else is done
DoneCompleted

Inbox

The Inbox is not a status — it's the set of tasks with no project assigned. Quick-captured tasks land here until you triage them into a project.

Assignees

Use the Assignee field to track who is responsible for a task. Assigned tasks appear in the assignee's view even if they don't own the project. Filter by assignee on the Kanban board or in the task list.

Priorities

Level Meaning
P0On fire — do it now, everything else waits
P1High — important, schedule soon
P2Medium — normal work

RULE OF THUMB: If everything is P0, nothing is P0. Use P0 only for actual fires. Most tasks should be P2.

Editing a Task

Click any task to open the detail view. Changes save automatically as you type (via Alpine.js). Fields include:

  • Title — required, shown everywhere
  • Description — rich notes (Markdown is stored but not rendered)
  • Status — dropdown to change workflow state
  • Priority — P0 through P2 or none
  • Start Date / Due Date — optional date range
  • Assignee — name of the person responsible
  • Project — which project this belongs to (or Inbox)
  • Blocked By — dependency on another task
  • Tags — comma-separated keywords
  • Links — external URLs related to this task
