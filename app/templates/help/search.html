SEARCH

TaskManager has two search engines running side by side:

Keyword Search (FTS5)

Always enabled. Searches task titles, descriptions, and tags using SQLite's full-text search. Fast, no dependencies.

Use the search bar in the header to find tasks by keyword.

Semantic Search (sqlite-vec)

Optional. Uses an AI embedding model (all-MiniLM-L6-v2 via ONNX) to understand the meaning of your search, not just exact words.

For example, searching "things I need to fix urgently" will find P0 tasks even if they don't contain those exact words.

NOTE: Semantic search is disabled by default because the embedding model adds ~65MB to the package. To enable it, set the environment variable EMBEDDING_ENABLED=true before starting the app.

Combined Search

When both are enabled, results are ranked by a combination of keyword match score and semantic similarity. The best matches appear first.
