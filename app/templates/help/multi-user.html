MULTI-USER SETUP

First-Time Setup

When you start TaskManager for the first time, you'll be taken to a setup wizard. Create your admin account and define your priority levels (default: P0, P1, P2). The admin user can see all tasks, projects, and logs across the entire system.

Logging In & Out

After setup, you'll log in with your username and password on every session. Your session is stored in a signed cookie — no database overhead. Click Logout in the header to end your session.

Admin vs Regular Users

TaskManager has two roles:

  • Admin — can see all tasks, projects, and logs. Can create and manage other users. Can share projects with team members.
  • Regular user — sees only their own items plus any projects explicitly shared with them. Can change their own password but cannot manage other users.

Project Sharing

Admins can share projects with specific team members from the project detail page. Look for the Share with section on a project's page.

  • Shared projects appear in the team member's task lists, dashboard, and search results.
  • If you can see a shared project, you can edit its tasks — there is no "view-only" mode.
  • Personal projects (not shared with anyone) are visible only to the owner — not even admins can see personal items.

Tasks Assigned to Other Users

If a task is assigned to a team member (via the Assignee field matching their username), they can see that task even if it's in a project they don't otherwise have access to.

TIP: TaskManager is designed for local network use — think shared office machine or small team on the same LAN. There's no OAuth, SSO, or email invites.
