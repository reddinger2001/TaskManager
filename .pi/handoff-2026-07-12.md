### Bug fix at end of session
- **Alpine.js x-data attribute quoting bug** — `tojson` produces double quotes inside `x-data="{...}"` HTML attributes, breaking the parser and leaking JS as visible text. Fixed by moving initial data into `<script type="application/json">` blocks and using Alpine's `init()` to load via `Object.assign()`. Applied to both task detail and project edit forms. **This fix is committed but NOT pushed to Forgejo yet** — needs a commit on `main`.

## Decisions made

- **[x-data via script tag]**: Never use `tojson` inside `x-data="..."` HTML attributes — double quotes break the parser. Use `<script type="application/json">` + `init()` + `Object.assign()` instead.
- **[Flask debug reloader doesn't watch templates]**: The `stat` reloader only restarts on `.py` file changes. Touch a Python file or restart manually after template edits.
- **[Port 5000 zombie processes]**: AirPlay Receiver can hold port 5000 on macOS. Use `lsof -ti:5000 | xargs kill -9` to clear it.
- **[Seed data idempotency]**: `db.create_all()` runs on every app startup. Never run seed scripts twice without dropping the DB first — duplicates accumulate silently.

## Things I think would make this better (hallucinated)

1. **Status transitions with a slide animation** — when dragging on the board, the card should slide into the new column instead of just appearing. Alpine.js `x-transition` can do this cheaply.

2. **Keyboard shortcuts** — `g d` for dashboard, `g b` for board, `g t` for tasks, `g p` for projects (like vim keybindings). The user is a terminal person — this would feel natural.

3. **Task capture from URL bar** — a browser extension or bookmarklet that captures the current page URL + title as a task. Security folks constantly need to "remember to look at this page later."

4. **"Someday/Maybe" inbox processing ritual** — a weekly digest view showing all inbox items with bulk actions (assign project, set priority, delete). Currently inbox items are just hidden behind a FAB toggle.

5. **Project-level health score** — computed from task completion %, overdue count, and P0 ratio. Display as a traffic-light indicator on the project card. Gives instant signal without opening the project.

6. **FTS5 for projects and logs** — currently only tasks have FTS5. Projects and logs use `ilike`. Adding FTS5 indexes would make search faster and support phrase matching across all content types.

7. **SQLite full-text search with stemming** — enable FTS5's built-in Porter stemmer so "monitoring" matches "monitored". Currently it's exact token matching only.

8. **Board filter persistence** — when you filter the board by project, the URL updates but navigating away and back loses the filter. Store in URL params (already done) + localStorage as a fallback for the default view.

9. **Dark mode toggle is unnecessary** — the sci-fi terminal aesthetic IS the dark mode. But consider a "light mode" for printing/exporting. The Gantt PDF will need this.

10. **Embedding model swap** — `all-MiniLM-L6-v2` is fast but mediocre. For a security domain, a fine-tuned model (e.g., BGE-small) would give much better semantic recall. Worth the 2x load time tradeoff for a personal tool.
