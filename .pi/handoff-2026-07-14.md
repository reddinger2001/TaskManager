# Handoff — 2026-07-14 — TaskManager: Test suite complete, code review done, 18 issues filed

## Where we are

TaskManager is a personal PM tool for a solo security engineer. Built on Flask + SQLite + sqlite-vec + FTS5 + Tailwind CSS (CDN) + Alpine.js (CDN), dark-mode sci-fi terminal aesthetic. Phase 1 and Phase 2 features are built out. The embedding model (sentence-transformers) crashes Python on page loads, so semantic search is disabled and all "Related Captures" sections are empty.

**This session:** Created a full pytest suite (165 tests, all passing), performed a thorough code review of the entire app, and filed 18 issues in Forgejo covering bugs, tech debt, refactors, and frontend improvements.

## Decisions made

- **Skip embedding hooks in tests** — the sentence-transformers model crashes Python. Tests skip `register_embedding_hooks` and `register_fts5_hooks`, manually recreate FTS5/vec0 virtual tables after `db.create_all()`.
- **Search for text parts, not full strings** — the highlight function wraps matches in `<mark>` tags, so searching for `"SOC Modernization"` fails because it's split into `"<mark>SOC</mark> Modernization"`. Tests search for individual words instead.
- **Check project card link, not name** — after deleting a project, checking that "SOC Modernization" doesn't appear in the list is wrong because child projects also contain that text. Check for the specific `href="/projects/1"` link instead.
- **Labels created on TaskManager repo**: bug (13), backend (11), frontend (12), refactor (8), tech-debt (9), quick-win (10).

## Blockers / open questions

- **Embedding model still crashes** — this is the #1 blocker for semantic search. The root cause (Python crash in sentence-transformers) was never diagnosed. Issue #38 addresses the symptom (blocking startup) but not the root cause.
- **No E2E tests** — the 165 pytest tests prove routes return correct HTML and DB state is correct, but they don't test Alpine.js at all. Board drag-and-drop, inline editing, modals, calendar rendering — none of these are verified in a real browser. Playwright E2E tests would be needed for that.
- **App may still be running on port 5000** from prior sessions — kill it: `lsof -ti:5000 | xargs kill -9`.
